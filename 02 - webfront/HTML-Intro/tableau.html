Les tableaux, c'est beau !

    Les tableaux s'utilisent dans les cas suivants :
  • pour retrouver rapidement et facilement des valeurs au croisement entre différents types de données.
  • pour des présentation des données tabulaires
  • pour l'organisation du contenu, puisque ces tableaux permettent de diviser visuellement une page Web en sections, ce qui facilite la lecture et la compréhension du contenu
  • pour les pages Web qui présentent de grandes quantités d'informations, car ils permettent aux utilisateurs de trouver rapidement ce qu'ils cherchent sans avoir à lire l'ensemble de la page.
  • pour afficher des informations complexes de manière claire et organisée, ce qui peut aider à maintenir l'attention des utilisateurs.
  • pour trier, filtrer et regrouper facilement les données présentées dans un tableau, ce qui peut aider les utilisateurs à visualiser les tendances et les modèles.
    Les tableaux ne sont pas recommandés dans les cas suivants :
  • pour l'accessibilité. Les tableaux de mise en page diminuent l'accessibilité aux malvoyants : les lecteurs d'écran, utilisés par les non-voyants, interprètent les balises d'une page HTML et lisent à haute voix le contenu à l'utilisateur.
  • pour la mise en page. Les tables produisent de la bouillie : Comme mentionné ci-dessus, les mises en page sur la base de tableaux comportent généralement des structures de balisage plus complexes que des techniques de mise en page appropriées. Le code résultant sera plus difficile à écrire, à maintenir et à déboguer.
  • pour la responsivité. Les tableaux ne s'adaptent pas automatiquement : Si vous utilisez les propriétés de mise en page (<header>,<section>, <article> ou <div>), leur largeur est par défaut 100% de celle du parent. Par contre, les tableaux sont dimensionnés en fonction de leur contenu par défaut, de sorte que des mesures supplémentaires sont nécessaires pour que le style du tableau fonctionne effectivement sur les différents types d'écran.
